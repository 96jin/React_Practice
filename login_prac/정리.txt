fs는 현재파일 기준이 아닌 실행하는 node콘솔 기준으로 경로를 바라본다.
fs.writeFile(경로, 파일) 은 경로의 파일에 변수에 있는 파일로 덮어쓴다.
readFile은 16진수의 buffer데이터로 읽어오기때문에 ,읽을 수 있는 JSON형식으로 변환해야한다.

함수는 한가지 기능만 수행하도록 해야한다.
Promise 로 만들어주지 않으면 하나의 함수에서 여러개의 작업을 동시에 수행하는 이도저도 아닌 코드가 만들어지게 된다.
이러한 비동기 작업을 처리해주기 위해서 async await, Promise 처리를 해준다.

클래스는 User 와 UserStorage 처럼 각자의 역할을 분명하게 구분시켜주는것이 좋다.
UserStorage에서는 DB 를 CRUD하는 역할만 수행하고, 
해당 데이터를 가지고 검증 및 조작하는 역할은 User가 수행하도록 구분.

웹서버와 WAS 의 차이 :
- 웹서버 : 클라이언트에서 요청이 오면 단순하게 서버에서 해당 정적 데이터를 전달만 해준다.

- WAS : 요청을 받아서 데이터를 가공한 후에 로직이 처리되고, 그것에 대한 응답을 전해줌

환경변수는 dotenv 로 관리할 수 있다.
dotenv 모듈을 불러와서 config() 하면 자동으로 .env에 등록되어있는 변수들을 접근할 수 있도록 해준다.
